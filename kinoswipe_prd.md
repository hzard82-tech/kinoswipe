# PRD: KinoSwipe — тиндер для фильмов

**Версия:** 1.0
**Дата:** 07.07.2026
**Автор:** Aaa
**Статус:** В разработке

---

## 1. Цель продукта

Персональное веб-приложение для быстрой сортировки топ-фильмов IMDB/TMDB в формате свайпов (по аналогии с Tinder), чтобы разобрать большой список фильмов на понятные категории: посмотреть, не интересно, понравилось, не понравилось. Не предназначено для продажи или публичного распространения — личный некоммерческий инструмент.

## 2. Проблема

Существует большой список известных/высокорейтинговых фильмов, но нет быстрого способа:
- отфильтровать, что уже смотрел и что нет;
- зафиксировать реакцию (понравилось / не понравилось) на просмотренное;
- сформировать список "хочу посмотреть" без ручного ведения таблиц.

Просмотр списка вручную (таблица, список закладок) медленный и неинтересный. Свайп-интерфейс ускоряет процесс и делает его вовлекающим.

## 3. Целевой пользователь

Один пользователь (сам автор), интерфейс не рассчитан на многопользовательский сценарий или публикацию.

## 4. Ключевые сценарии использования

1. Пользователь открывает сайт и видит карточку фильма (постер, название, год, рейтинг, жанры).
2. Свайп **вправо** → "хочу посмотреть".
3. Свайп **влево** → "не интересно / не буду смотреть".
4. Свайп **вверх** → "уже посмотрел, понравилось".
5. Свайп **вниз** → "уже посмотрел, не понравилось".
6. Решение сохраняется, показывается следующая карточка.
7. Пользователь может открыть вкладки с результатами по каждой категории.
8. Пользователь может отменить последний свайп или сбросить весь прогресс.
9. При повторном визите прогресс не теряется (сохранён в браузерном/встроенном хранилище).

## 5. Источник данных о фильмах

- **Основной источник:** TMDB API (`/movie/top_rated`), т.к. у IMDB нет публичного бесплатного API.
- Пользователь предоставляет собственный бесплатный API-ключ TMDB (зарегистрирован для личного некоммерческого использования).
- Загружаются: название, год выхода, рейтинг, количество голосов, постер, жанры — на русском языке (`language=ru-RU`).
- Целевой объём: **до 1000 фильмов** (50 страниц по 20 фильмов из `top_rated`).
- Известное ограничение: рейтинг TMDB "top rated" — не то же самое, что рейтинг IMDB Top 1000; это ближайший доступный аналог с реальными данными.

## 6. Технические ограничения окружения (важно)

Приложение — React-артефакт в песочнице Claude.ai, что накладывает жёсткие ограничения:

| Ограничение | Следствие |
|---|---|
| CSP может блокировать `fetch`/XHR к сторонним доменам из браузерной песочницы артефакта | Запрос к TMDB может не выполниться даже с валидным ключом — требует проверки в реальном окружении |
| `localStorage` / `sessionStorage` запрещены | Используется встроенный `window.storage` (личное хранилище артефакта) |
| Нет собственного бэкенда | Все запросы идут напрямую из браузера пользователя к TMDB |

**Если живой запрос к TMDB не работает в артефакте:** предусмотреть запасной вариант — встроенный (baked-in) список фильмов без обращения к внешнему API, с стилизованными карточками без реальных постеров.

## 7. Функциональные требования

### 7.1 Загрузка данных
- Экран ввода API-ключа TMDB при первом запуске.
- Прогресс-бар загрузки (0–100%) на время сбора ~1000 фильмов.
- Кэширование каталога фильмов во встроенном хранилище (общее/shared), чтобы не грузить повторно.
- Обработка ошибок: неверный ключ, сетевая ошибка — понятное сообщение с инструкцией.

### 7.2 Свайп-интерфейс
- Карточка: постер, название, год, рейтинг (звезда), количество голосов, до 3 жанров.
- Поддержка свайпа мышью/тачем (drag) в 4 направлениях.
- Поддержка стрелок клавиатуры (← → ↑ ↓).
- Кнопки на экране как альтернатива свайпу (для доступности / десктопа без мыши-драга).
- Визуальная подсказка направления во время перетаскивания (цвет + подпись).
- Анимация возврата карточки, если свайп не дотянул до порога.

### 7.3 Категории и списки
- 4 категории: "буду смотреть" (право), "не интересно" (влево), "понравилось" (вверх), "не понравилось" (вниз).
- Вкладки с счётчиками по каждой категории.
- Просмотр списка по категории (сетка постеров + мета).
- Возможность вернуть фильм обратно в очередь свайпа ("оценить заново").

### 7.4 Управление прогрессом
- Отмена последнего свайпа (undo).
- Полный сброс всех оценок с подтверждением.
- Счётчик "оценено X из 1000" в шапке.
- Экран завершения, когда очередь пуста.

## 8. Дизайн

- Стиль: тёмная кинозальная тема (почти чёрный фон), акценты — золото (`#D4AF37`) и бордовый (`#B23A4E`/тёмно-красный для "не понравилось").
- Типографика: постерный шрифт (Bebas Neue) для заголовков, Inter — основной текст, JetBrains Mono — цифры/счётчики.
- Фирменный элемент: карточка стилизована под кадр киноплёнки — перфорация (сквозные отверстия) по левому и правому краю.
- Без шаблонных элементов (не крем/терракота, не дефолтный тёмный + один яркий акцент без обоснования).

## 9. Хранение данных

| Ключ | Тип | Назначение |
|---|---|---|
| `kino-catalog-v1` | shared | Список фильмов + жанры + время загрузки |
| `kino-swipes-v1` | personal | Решения пользователя по каждому фильму (id → направление, время) |

## 10. Нефункциональные требования

- Полностью клиентское приложение, без сервера.
- Отзывчивый интерфейс на мобильных экранах.
- Видимый фокус при навигации с клавиатуры.
- Не собирает и не передаёт данные пользователя куда-либо, кроме встроенного хранилища артефакта и API TMDB (только для получения данных о фильмах).

## 11. Открытые вопросы / риски

- **Риск:** живой запрос к TMDB может быть заблокирован CSP-политикой артефакта — требует практической проверки в реальном чате.
- **Запасной план:** встроенный статический список фильмов (без реальных постеров, стилизованные карточки) — обсуждался как альтернатива, но пока не реализован по решению пользователя (выбран вариант с реальным API-ключом).
- Точность соответствия "TMDB top rated" и "настоящего топ-1000 IMDB" — не идентичны по составу и рейтингам.

## 12. Вне рамок (Out of scope)

- Многопользовательский режим, публикация в открытый доступ.
- Монетизация, реклама.
- Рекомендательная система на основе истории свайпов (в текущей версии).
- Интеграция с трекерами вроде Кинопоиска/Letterboxd.

## 13. Метрики успеха

Продукт личный, формальных бизнес-метрик нет. Критерий успеха — пользователь может:
1. За один присест разобрать значительную часть из 1000 фильмов;
2. Найти позже свои категории без потери прогресса;
3. Пользоваться сайтом без раздражающих сбоев (загрузка, свайп-жесты).
